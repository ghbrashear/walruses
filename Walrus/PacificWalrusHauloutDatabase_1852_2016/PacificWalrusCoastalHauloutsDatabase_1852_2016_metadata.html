Description:
Abstract:
We summarize available information on Pacific walrus haulouts from available reports, interviews with coastal residents and aviators, and personal observations of the authors. We provide this in the form of a georeferenced database that may be queried and displayed with standard geographic information system and database management software. The database contains 150 records of Pacific walrus haulouts, with a summary of basic characteristics on maximum haulout size, age-sex composition, season of use, and decade of most recent use. Citations to reports are provided as a bibliographic database.
Purpose:
These data were compiled to provide resource managers with a consolidated and comprehensive source of information about location, history and summary characteristics of reported Pacific walrus haulouts throughout their range.
Supplemental_Information:
The database consists of a set of interrelated digital files, including the geographic representation of the haulouts and a bibliographic database of the reports cited for each haulout record. The georeferenced component of the database is provided in two digitally accessible forms: one intended for display and query in a Geographic Information System (GIS), and another for display and query in a virtual globe. The form of the database intended for display and query in a GIS is provided in two file formats: a tabular comma separated value (CSV) file (PacificWalrusHauloutDatabase.csv) and a shape file (PacificWalrusHauloutsDatabaseLinearFeatures.shp). The CSV file is encoded using the UTF-8 character encoding and can include all fields of the database, including the Cyrillic script field containing Russian language names of the haulouts. The shape file contains a geographic representation of spatial lines representing haulouts that are depicted as linear features. Linear features contained in the shape file may be linked back to the full record of the haulout contained within the CSV file based on a key field. The form of the database intended for display and query in a virtual globe is provided as a single keyhole markup language (KML) file (PacificWalrusHauloutDatabase.kml) that contains all points and linear features, including the Cyrillic script field with the Russian language names of the haulouts. The KML file can be viewed in a virtual globe such as Google Earth (http://www.google.com/earth/), Marble (http://marble.kde.org/), or World Wind (http://worldwind.arc.nasa.gov/java/). The bibliographic component of the database (the file PacificWalrusHauloutDatabaseBibliography_DS.RIS) provides a listing of all reports cited in the database. This machine readable bibliographic data file is a complete copy of the citations listed in appendix A of the summary report.

Taxonomic_Procedures:
Taxonomy of the study species was accepted from the available reports, personal communications and personal observations with the assurance that the only walrus sub-species observed to date within the study area is the Pacific walrus. Aerial observations of Pacific walrus coastal haulouts were only accepted if the observers clearly indicated that the observations could not be confused with coastal haulouts of sea lions, or harbor and ice seals.

Horizontal_Positional_Accuracy:
Horizontal_Positional_Accuracy_Report:
We assigned a geographic location to each haulout based on available data sources, and adjusted the location based on publicly available geo-referenced imagery (Digital Globe, Longmont, Colorado, USA, viewed through the virtual digital globe geographical information system Google Earth TM ). Spatial accuracy varies among the haulout records, but is generally within 5 kilometers of the actual reported haulout location.

Methodology_Description:
Data on Pacific haulout locations and their historical use was compiled from available data sources and entered into a Microsoft Access database.
Process_Step:
Process_Description:
We exported a comma separated value file encoded in UTF-8 that contained all fields within the database. We used the plotKML contributed package in r (plotKML v 0.5-6; http://cran.r-project.org/web/packages/plotKML accessed 16 April 2016) to construct a Keyhole Markup Language file from the database with all fields including the Cyrillic script field containing the Russian language names of the haulout locations. We used the ESRI software ArcGIS (v 10.1) to digitize coastal linear features represented haulouts that occurred along an extended coastline.

Overview_Description:
Entity_and_Attribute_Overview:
The database consists of a set of interrelated digital files (PacificWalrusHauloutDatabase.csv, PacificWalrusHauloutsDatabaseLinearFeatures.shp, PacificWalrusHauloutDatabase.KML, and PacificWalrusHauloutDatabaseBibliography_DS.RIS). The georeferenced component of the database is provided in two digitally accessible forms: one intended for display and query in a Geographic Information System (GIS), and another for display and query in a virtual globe. The form of the database intended for display and query in a GIS is provided in two file formats: a tabular comma separated value or CSV file (PacificWalrusHauloutDatabase.csv) and a shape file (PacificWalrusHauloutsDatabaseLinearFeatures.shp). The CSV file is encoded using the UTF-8 character encoding and can include all fields of the database, including the Cyrillic script field containing Russian language names of the haulouts. The shape file contains a geographic representation of spatial lines representing haulouts that are depicted as linear features. Linear features contained in the shape file may be linked back to the full record of the haulout contained within the CSV file based on a key field.

The form of the database intended for display and query in a virtual globe is provided as a single keyhole markup language or KML file (PacificWalrusHauloutDatabase.KML) that contains all points and linear features, including he Cyrillic script field with the Russian language names of the haulouts. The KML file can be viewed in a virtual globe such as Google Earth (http://www.google.com/earth/), Marble (http://marble.kde.org/), or World Wind (http://worldwind.arc.nasa.gov/java/).

In addition to the haulout geographic database files, we provided a machine-readable bibliographic file that is a full copy of the references cited in the database (n = 151, PacificWalrusHauloutDatabaseBibliography_DS.RIS). Bibliographic data management software is required to display and manipulate the citations in the machine readable bibliographic file. However, a full listing of these references is provided in a standard bibliographic listing format in Appendix A of the USGS Open File Report (USGS OFR 2016-110, http://dx.doi.org/10.3133/ofr20161108).
